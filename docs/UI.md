# UI

## Web 控制台

- 页面入口：`/ui`
- 当前仍为原生 `HTML + CSS + JS` 方案
- 静态资源目录：`apps/api/src/modules/ui/public`
- 页面由 API 服务直接托管，不依赖单独的前端构建产物
- 投放执行表卡片固定展示精简后的字段预览，不再允许前端拼装技术列；其中 `投放项名称` 会作为飞书主字段固定在第一列，`七天后数据` 明确标注为系统自动回填

## 全局 AI 悬浮入口

- 位置：页面右下角
- 名称：`Guru Ads Agent`
- 目标：为未来的 AI 能力提供统一入口，而不是把单个模型按钮散落在页面各处
- 当前形态：
  - 收起时为全局悬浮球
  - 展开时为右侧 modal drawer 对话窗
  - 顶部提供模型下拉，按当前可用 provider 动态展示
  - 当前支持：`Qwen 3.6-Plus`、`Kimi-K2.5 (OpenRouter)`、`GPT-5.4 (OpenAI)`
  - 面板内保留 Gemini 官网外部工具入口

### 当前交互

- 点击悬浮球：展开 / 收起 `Guru Ads Agent`
- 点击面板外区域：自动收起
- 按 `Esc`：
  - 若工具面板已展开，先收起工具面板
  - 否则关闭整个抽屉
- 切换模型：
  - 默认优先选择当前有凭据的主模型
  - 会记住本地上次选择
  - 若当前模型不可用或不支持图片，会在发送前或发送后给出明确提示
- 点击 `打开 Gemini 官网`：新标签页跳转到 `https://gemini.google.com/`
- 直接自然语言提问时：
  - 如果是明确的数据型业务问题，Agent 会优先结合当前页面上下文自动调用内部只读工具
  - 如果参数仍不够，会进入 `clarification` 追问，而不是直接报错

### 当前能力

- 页面内多轮对话
  - 刷新页面后会清空会话
- 模型能力识别
  - 支持从 `/api/ai/models` 读取 provider、模型名、是否支持图片 / thinking
- 自然语言自动查库
  - 默认携带 `page_context_json`
  - 当前自动工具范围：
    - `apps.list`
    - `metrics.get_trend`
    - `roas.get_summary`
    - `budget.get_summary`
    - `asa_keywords.get_summary`
  - 回复下方会显示简洁 `tool_trace`，例如“已自动查询：预算建议摘要”
  - 当用户问“ROAS / 回收 / D7 ROAS”，且语义更接近日报 / 简报口径时，Agent 会优先使用 `roas.get_summary`
    - 通用每日简报 / 预算建议走 `scope=budget`
    - ASA 简报 / ASA 看板走 `scope=asa`
    - 回答中应明确写出 `报告日期` 与 `成熟窗口 from ~ to`
    - 这不是最新单日实时 ROAS，而是成熟窗口聚合值
- 图片上传
  - 最多 4 张
  - 支持 `png/jpeg/webp`
  - 单张最大 5MB
  - 若当前模型不支持图片，会提示切回支持图片的模型或移除图片
- 数据库聚合上下文包
  - 当前模块推荐
  - 核心数据包
  - 自定义模板
  - 手动附加的数据包优先级高于自动查询
  - `roas_summary` 若手动指定 `reportDate`，会原样透传到后端，不会被默认“昨天”覆盖
- 外部工具
  - 当前保留 Gemini 官网快捷入口

### 当前使用约束

- 模型下拉展示的是“当前已配置且可用的模型”，不是 provider 能力的绝对承诺
  - 例如 `Kimi-K2.5 (OpenRouter)` 可能在某些账号或地区下仍然无法稳定处理图片
- 自动查库依赖内部 `mcp-server`
  - 若 MCP 查询超时或不可用，界面通常会表现为 warning、timeout 或 provider 失败，而不是静默回退
- 指标趋势页默认时间窗按浏览器本地时间生成
  - `pull` 走本地日期 `YYYY-MM-DD`
  - `push` 走本地日期时间 `YYYY-MM-DD HH:mm:ss`
  - 凌晨时段不会再因为 UTC `toISOString()` 提前错到前一天
- 当前页面内的问号帮助以点击 / hover 问号按钮为主，不建议把整块术语文字当作可触发热区

## ASA 关键词管理补充

- `CPP` 与 `D7 ROAS` 不是最新单日即时值，而是按成熟窗口汇总后的摘要值
- 当前所有用户可见的 `ROAS / D7 ROAS` 均统一显示为百分比
  - 例如存储层的 `1.5` 会显示成 `150.00%`
- 页面会根据 `roas_data_status` 区分 5 种状态：
  - `complete`：成熟窗口数据完整，可直接使用
  - `partial`：成熟窗口覆盖率达到阈值，但仍有缺口；当前值按已覆盖成本计算
  - `partial_low`：成熟窗口覆盖率偏低，当前值仅供参考，不直接驱动动作
  - `pending`：成熟窗口覆盖率仍不足，UI 会显示“待补齐（源数据缺失）”
  - `unavailable`：当前还没有可用于判断的成熟窗口
- 因此即便部分 keyword 已经拿到收入事件，只要成熟窗口总覆盖率还没达到阈值，摘要卡里的 `CPP / D7 ROAS` 仍可能统一显示“待补齐（源数据缺失）”

### 工具面板结构

- 一级分组：
  - `数据库上下文`
  - `图片附件`
  - `外部工具`
- `数据库上下文` 下的二级折叠：
  - `当前模块推荐`
  - `核心数据包`
  - `自定义模板`
- 打开工具面板时，抽屉主体优先展示工具内容，避免被消息区或输入区遮挡

### 设计约束

- 视觉语言需与当前投放工作台保持统一
  - 延续浅色控制台、青蓝强调色、轻磨砂卡片、柔和阴影
- 交互优先级高于特效
  - 工具面板、消息区、输入区不能互相遮挡
- 悬浮球允许存在识别性动效
  - 但不能破坏工作台整体的稳定感与信息密度
- 移动端需要自动收敛
  - 小屏下保留浮球主体，弱化附属文案
  - 抽屉需要切换为更明确的全高侧滑面板

### 后续扩展建议

- 保持一个统一的 AI 入口容器
- 新功能优先接入面板内，而不是直接新增新的全局浮层
- 当前已具备自然语言自动查库能力；后续建议优先扩展为：
  - 诊断助手
  - 投放 Copilot
  - 日报快读 / 总结
  - 告警解释与排查建议
