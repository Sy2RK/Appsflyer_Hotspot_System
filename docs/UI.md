# UI

## Web 控制台

- 页面入口：`/ui`
- 当前仍为原生 `HTML + CSS + JS` 方案
- 静态资源目录：`apps/api/src/modules/ui/public`
- 页面由 API 服务直接托管，不依赖单独的前端构建产物

## 全局 AI 悬浮入口

- 位置：页面右下角
- 目标：为未来的 AI 能力提供统一入口，而不是把单个模型按钮散落在页面各处
- 当前形态：
  - 收起时为全局悬浮球
  - 展开时为“未来功能舱”面板
  - 当前只提供 Gemini 官网跳转

### 当前交互

- 点击悬浮球：展开 / 收起功能舱
- 点击面板外区域：自动收起
- 按 `Esc`：收起
- 点击 `打开 Gemini 官网`：新标签页跳转到 `https://gemini.google.com/`

### 设计约束

- 视觉语言需与当前投放工作台保持统一
  - 延续浅色控制台、青蓝强调色、轻磨砂卡片、柔和阴影
- 动效可以强化“AI 入口”的辨识度
  - 但不能破坏工作台整体的稳定感与信息密度
- 移动端需要自动收敛
  - 小屏下保留浮球主体，弱化附属文案

### 后续扩展建议

- 保持一个统一的 AI 入口容器
- 新功能优先接入面板内，而不是直接新增新的全局浮层
- 若未来接入真实对话能力，建议优先扩展为：
  - 诊断助手
  - 投放 Copilot
  - 日报快读 / 总结
  - 告警解释与排查建议
