预算建议

预算动作与执行动作说明

  • 核心口径：优先使用 AppsFlyer 广告日报返回的官方 eCPI。
  • 低量级：最近 3 天激活量 < 15，默认观察，不自动调价。
  • 中量级与高量级：是否上调、下调或暂停，仍由核心指标和阈值规则决定。
  • 预算调幅不再固定写死，可按应用级规则单独配置默认上调、默认下调和高花费扩量比例。
  • 低量级且信号偏弱时，系统会优先给出“保持预算 + 迭代素材 / 提升跑量能力”的执行动作。
  • 高量级且持续上涨时，如果主动作是扩量，系统会叠加“提高 ROAS 目标 / 稳步扩量”的执行动作。
  • 目标 eCPI：优先按同应用、同平台、同媒体源的有效关键词 P40 计算，再用同平台中位数做上限钳制。
  • 极端低效且已进入 pause_candidate 的关键词，才会给出暂停建议。

这是给调控师参考的预算动作建议。系统不会自动执行，需要人工确认后再处理。 每次安装成本（eCPI） 每次安装成本（eCPI） 表示平均花多少钱带来一个安装。越低通常代表买量效率越好，越高代表成本偏贵。
